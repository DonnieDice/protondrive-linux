# Documentation

## Purpose

Project documentation including API docs, architecture diagrams, development guides, and templates.

## Directory Structure

```
docs/
├── api/                # API documentation
├── architecture/       # Architecture diagrams and docs
├── development/        # Development guides
├── guides/             # How-to guides
└── templates/          # Documentation templates
```

## Documentation Types

### API Documentation (`api/`)
API reference documentation for services and utilities.

### Architecture (`architecture/`)
System architecture diagrams, design decisions, and technical specifications.

### Development (`development/`)
Development setup guides, coding standards, and contribution guidelines.

### Guides (`guides/`)
Step-by-step guides for common tasks:
- Creating directory READMEs
- Adding new services
- Writing tests

### Templates (`templates/`)
Reusable templates for documentation:
- Directory README template
- ADR (Architecture Decision Record) template
